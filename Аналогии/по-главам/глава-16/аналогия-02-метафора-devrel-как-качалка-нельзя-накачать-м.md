# Метафора: DevRel — как качалка. Нельзя накачать мышцы за неделю, но за год регулярных тренировок результат будет виден всем. Яндекс качал "мышцы репутации" 10+ лет. Результат? Люди сами хотят к ним работать.

**Контекст**: Глава 16


**Конференции и события:**
- **Яндекс.Фронтенд** — ежегодная конференция по frontend-разработке (2,000+ участников)
- **Я.Субботник** — конференция по backend и инфраструктуре (1,500+ участников)  
- **Mobius** — мобильная разработка (800+ участников)
- **YaC (Yet another Conference)** — общетехническая конференция (3,000+ участников)
- Регулярность: 2-4 крупных события в год

**Митапы и локальные сообщества:**
- Регулярные митапы по технологиям в офисах (Москва, СПб, Екатеринбург, Новосибирск)
- Частота: 8-12 митапов в месяц по всей России
- Средняя аудитория: 50-150 человек
- Темы: React, Python, Go, ML (машинное обучение), DevOps (разработка и эксплуатация), QA (тестирование)

**Контент-стратегия:**
- **Habr корпоративный блог:** 2-3 статьи в неделю, 150+ статей в год
- **YouTube канал:** 50,000+ подписчиков, записи докладов и туториалы
- **Подкасты:** "Пятиминутка React", "Без СМС и регистрации"
- **Технический блог:** deep-dive статьи по архитектуре и алгоритмам

**Open Source и developer tools:**
- **ClickHouse** — аналитическая СУБД (25,000+ звезд на GitHub)
- **YDB** — распределенная база данных
- **CatBoost** — библиотека машинного обучения (ML)
- **Arcadia** — система сборки (аналог Bazel)
- **Множество библиотек:** от UI (пользовательского интерфейса) компонентов до ML (машинного обучения) инструментов

**Сообщество и каналы:**
- Telegram каналы по технологиям (общая аудитория 100,000+)
- GitHub организации с открытыми проектами
- Участие в международных конференциях (React Conf, PyCon, etc.)
