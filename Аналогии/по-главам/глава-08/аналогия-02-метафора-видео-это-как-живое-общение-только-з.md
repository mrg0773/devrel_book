# Метафора: Видео — это как живое общение, только записанное. Представьте, что вы объясняете коллеге как настроить Docker. Вот это и есть хорошее техническое видео.

**Контекст**: Глава 8


**Форматы:**

**Короткие видео (5-10 минут):**
- Quick tips
- Tool reviews
- News и обновления

**Средние (15-30 минут):**
- Туториалы
- Code walkthroughs
- Tech talks записи

**Длинные (1+ час):**
- Мастер-классы
- Livestreams (coding sessions)
- Подробные курсы

**Площадки:**
- YouTube (основная)
- Twitch (для livestreams)
- VK Видео (если российская аудитория)

**Преимущества видео:**
- Показываете процесс (не только результат — это важно!)
- Личный контакт (люди видят лицо, слышат голос)
- Легче воспринимается чем текст (для некоторых — не для всех)

**Недостатки:**
- Дороже в production (камера, свет, монтаж — это не бесплатно)
- Сложнее обновлять (текст поправил за 5 минут, видео — переснимай)
- Требует навыков (или денег на специалиста)
